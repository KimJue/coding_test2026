# 📚 Coding Test Algorithm Summary

코딩테스트에서 자주 등장하는 알고리즘 유형 정리

---

# 1️⃣ Hash (해시)

## 핵심 개념
빠르게 탐색 / 중복 제거 / 개수 세기

시간복잡도 대부분 `O(1)`

---

## ✅ HashSet

### 사용 목적
- 중복 제거
- 존재 여부 확인

### 사용 예시

```java
HashSet<Integer> set = new HashSet<>();
```

### 문제에서 보이는 힌트
- "중복 제거"
- "이미 존재하는가?"
- "서로 다른 개수"

### 대표 문제 유형
- 방문 체크
- 중복 문자열 제거
- 서로 다른 숫자 개수

---

## ✅ HashMap

### 사용 목적
- 개수 세기
- key-value 저장
- 빈도 계산

### 사용 예시

```java
HashMap<String, Integer> map = new HashMap<>();
```

### 문제에서 보이는 힌트
- "몇 번 등장?"
- "빈도수"
- "카운트"

### 대표 문제 유형
- 완주하지 못한 선수
- 의상
- 베스트앨범

---

# 2️⃣ Stack / Queue

# ✅ Stack

## 특징
후입선출 (LIFO)

```java
Stack<Integer> stack = new Stack<>();
```

## 언제 사용?
- 최근 데이터 먼저 처리
- 괄호 검사
- 되돌리기(Undo)

## 문제 힌트
- "() 올바른 괄호"
- "최근 작업 취소"
- "역순 처리"

## 대표 문제
- 올바른 괄호
- 탑
- 주식가격

---

# ✅ Queue

## 특징
선입선출 (FIFO)

```java
Queue<Integer> q = new LinkedList<>();
```

## 언제 사용?
- 순서대로 처리
- BFS
- 대기열 처리

## 문제 힌트
- "먼저 들어온 순서"
- "차례대로 처리"
- "최단거리"

## 대표 문제
- 프로세스
- 다리를 지나는 트럭
- BFS 문제

---

# 3️⃣ Heap (힙 / PriorityQueue)

## 핵심 개념
최대값/최소값 빠르게 추출

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
```

---

## 언제 사용?
- 최소값 반복 추출
- 최대값 반복 추출
- 우선순위 처리

## 문제 힌트
- "가장 작은 값"
- "최댓값 반복"
- "우선순위"

## 대표 문제
- 더 맵게
- 디스크 컨트롤러
- 야근 지수

---

# 4️⃣ Sort (정렬)

## 핵심 개념
정렬 후 규칙 찾기

```java
Arrays.sort(arr);
```

---

## 언제 사용?
- 크기 비교
- 순서 기반 처리

## 문제 힌트
- "오름차순"
- "내림차순"
- "정렬"

## 대표 문제
- K번째 수
- 가장 큰 수
- H-index

---

# 5️⃣ Brute Force (완전탐색)

## 핵심 개념
가능한 경우를 모두 탐색

---

## 언제 사용?
- 입력 크기가 작음
- 모든 경우 확인 가능

## 문제 힌트
- "모든 경우"
- "가능한 조합"
- "브루트포스"

## 대표 문제
- 모의고사
- 카펫
- 소수 찾기

---

## 자주 사용하는 방식

### 순열 (Permutation)
순서 중요

### 조합 (Combination)
순서 중요 X

### 비트마스킹
부분집합 탐색

---

# 6️⃣ Greedy (탐욕법)

## 핵심 개념
현재 상황에서 가장 좋은 선택

---

## 언제 사용?
- 현재 최선이 전체 최선으로 이어짐

## 문제 힌트
- "최대한"
- "최소 횟수"
- "가장 큰 값"

## 대표 문제
- 체육복
- 큰 수 만들기
- 구명보트

---

## 핵심 포인트
정렬과 함께 자주 사용됨

---

# 7️⃣ Dynamic Programming (DP)

## 핵심 개념
이전 계산 결과를 저장해서 재사용

---

## 언제 사용?
- 같은 계산 반복 발생
- 이전 결과 활용 가능

## 문제 힌트
- "최댓값"
- "최소 비용"
- "경우의 수"

## 대표 문제
- 피보나치 수열
- 정수 삼각형
- N으로 표현

---

## DP 기본 형태

```java
dp[i] = dp[i - 1] + dp[i - 2];
```

---

# 8️⃣ DFS / BFS

# ✅ DFS (깊이 우선 탐색)

## 특징
끝까지 먼저 탐색

```java
재귀 or Stack 사용
```

## 언제 사용?
- 모든 경로 탐색
- 백트래킹

## 문제 힌트
- "모든 경우 탐색"
- "가능한 경로"

## 대표 문제
- 타겟 넘버
- 네트워크
- 여행경로

---

# ✅ BFS (너비 우선 탐색)

## 특징
가까운 곳부터 탐색

```java
Queue 사용
```

## 언제 사용?
- 최단거리 탐색
- 레벨 단위 탐색

## 문제 힌트
- "최소 이동"
- "최단 거리"

## 대표 문제
- 게임 맵 최단거리
- 미로 탐색

---

# DFS vs BFS

| DFS | BFS |
|---|---|
| 깊게 탐색 | 가까운 곳부터 |
| Stack/재귀 | Queue |
| 모든 경우 탐색 | 최단거리 |
| 백트래킹 | 거리 계산 |

---

# 9️⃣ Binary Search (이분탐색)

## 핵심 개념
탐색 범위를 절반씩 줄이기

---

## 언제 사용?
- 정렬된 데이터
- 빠른 탐색 필요

## 문제 힌트
- "최대 길이"
- "가능한 값"
- "logN"

## 대표 문제
- 입국심사
- 랜선 자르기
- 공유기 설치

---

## 기본 형태

```java
while(left <= right) {
    int mid = (left + right) / 2;
}
```

---

## 핵심 개념
노드와 간선의 연결 관계 탐색

---

## 언제 사용?

## 문제 힌트
- "연결"

## 대표 알고리즘

- 다익스트라

# 🚨 문제를 보고 알고리즘 판단하는 법

# 1. 키워드 보기

| 키워드 | 의심 알고리즘 |
|---|---|
| 중복 제거 | HashSet |
| 빈도수 | HashMap |
| 최단거리 | BFS |
| 모든 경우 | DFS / 완전탐색 |
| 최소 비용 | DP / Greedy |
| 가장 큰/작은 | Heap |
| 정렬 | Sort |
| 빠른 탐색 | Binary Search |

---

# 2. 입력 크기 확인

| 입력 크기 | 추천 알고리즘 |
|---|---|
| N ≤ 10 | 완전탐색 |
| N ≤ 1000 | O(N²) 가능 |
| N ≥ 100000 | O(N log N) 필요 |

---

# 3. 자주 나오는 패턴

| 문제 패턴 | 알고리즘 |
|---|---|
| 개수 세기 | HashMap |
| 중복 제거 | HashSet |
| 괄호 검사 | Stack |
| 최단거리 | BFS |
| 경우의 수 | DFS |
| 최소/최대 반복 | Heap |
| 이전 결과 활용 | DP |

---

# ✅ 코딩테스트 핵심

문제를 읽고 아래를 빠르게 판단하는 연습이 중요하다.

- 중복인가?
- 개수 세기인가?
- 최단거리인가?
- 모든 경우를 탐색해야 하는가?
- 이전 계산을 재사용할 수 있는가?

이 감각이 생기면 알고리즘 유형이 빠르게 보이기 시작한다.
